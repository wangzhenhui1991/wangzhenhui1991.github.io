发表于 2016-08-31   |   分类于 Linux   |  

磁盘空间/dev/sdb2-100%

背景：在VMWare12中使用OpenSUSE13.2的过程中，在一次练习Hadoop中发现系统崩溃
时间：2016-8-30
虚拟机：VMWare-12
系统：OpenSUSE-13.2
问题：逻辑分区/dev/sdb2 可用空间0%(10.9GB/11GB)
导致：程序无法启动，bash无法使用，kde自动退出，系统进而退出。

解决过程：
几个查看命令：
$ df -h #查看磁盘空间
$ ls -Sl | more -5 #按大小排序详细显示文件信息，并以每5个展现
$ du -s -h ./* #查看目录占用情况并显示大小

跟踪查看了/目录下的文件大小，发现主要是

  • /tmp—->/var/tmp
    /tmp目录实际链接到/var/tmp缓存文件，里面有大量目录缓存，zypper的安装缓存，以及VMWareDnd(即客户机到虚拟机拖拽传送的文件缓存)，占据了大概700M的内容

  • /usr/
    /usr中主要是/lib和/lib64这两个存放各种库文件的目录，随着安装的程序增多，会占据越来越多的控件，占了大概2-3G的空间，不太好清理

  • /log
    这里面存放着系统的日志文件

$ zypper install tmpwatch #安装tmpwatch
$ tmpwatch 24 /tmp #删除/tmp文件夹中24小时以上没有使用的文件

Hadoop

Hadoop就是解决了大数据（大到一台计算机无法进行存储，一台计算机无法在要求的时间内进行处理）的可靠存储和处理。

Apache Hadoop 简介

Apache Hadoop 是一个开源软件框架，可安装在一个商用机器集群中，使机器可彼此通信并协同工作，以高度分布式的方式共同存储和处理大量数据。最初，Hadoop 包含以下两个主要组件：Hadoop Distributed File System (HDFS) 和一个分布式计算引擎，该引擎支持以 MapReduce 作业的形式实现和运行程序。

MapReduce 是 Google 推广的一个简单的编程模型，它对以高度并行和可扩展的方式处理大数据集很有用。MapReduce 的灵感来源于函数式编程，用户可将他们的计算表达为 map 和 reduce 函数，将数据作为键值对来处理。Hadoop 提供了一个高级 API 来在各种语言中实现自定义的 map 和 reduce 函数。

Hadoop 还提供了软件基础架构，以一系列 map 和 reduce 任务的形式运行 MapReduce 作业。Map 任务 在输入数据的子集上调用 map 函数。在完成这些调用后，reduce 任务 开始在 map 函数所生成的中间数据上调用 reduce 任务，生成最终的输出。 map 和 reduce 任务彼此单独运行，这支持并行和容错的计算。

最重要的是，Hadoop 基础架构负责处理分布式处理的所有复杂方面：并行化、调度、资源管理、机器间通信、软件和硬件故障处理，等等。得益于这种干净的抽象，实现处理数百（或者甚至数千）个机器上的数 TB 数据的分布式应用程序从未像现在这么容易过，甚至对于之前没有使用分布式系统的经验的开发人员也是如此。

java-多线程

发表于 2016-07-17   |   分类于 java ， 并发 ， 多线程   |  

概述 程序-进程-线程

程序 Program,是一个静态的概念
进程 Process,是一个动态的概念,进程是程序的一次动态执行过程, 占用特定的地址空间.每个进程都是独立的，由3部分组成cpu,data,code,缺点：内存的浪费，cpu的负担
线程 Thread,是进程中一个“单一的连续控制流程”.又被称为轻量级进程(lightweight process),一个进程可拥有多个并行的(concurrent)线程,一个进程中的线程共享相同的内存单元/内存地址空间可以访问相同的变量和对象，而且它们从同一堆中分配对象通信、数据交换、同步操作,由于线程间的通信是在同一地址空间上进行的，所以不需要额外的通信机制，这就使得通信更简便而且信息传递的速度也更快。
可以从jvm的内存结构上来理解线程与进程的区别：
jvm中每个线程拥有自己独立的栈空间stack，程序计数器counter，本地方法栈native等，
而整个进程即可以的内存空间则主要是指jvm中的堆空间和方法区。
